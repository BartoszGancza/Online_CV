My projects:

Tweezel

Simple Twitter application created as a graduation project for online programming course CS50x by Harvard.

Key features:
  • Search for any user to check their public account information and see when they are most active (based on 100 recent Tweets)
  • Post a Tweet - including emoji
  • View your 30 newest followers
  • Check your timeline, responses and direct messages

Technology outline:

Flask was used to handle client/server and Twitter API interactions as well as to manage users session.

Twitter API calls were made through one of the Twitter's API wrappers for Python called Twython. Users log in through Twitter's own authentication page and the resulting tokens are stored in a session file (both app and user based authentication is handled at the same time).

Most of the application content is dynamically generated by first fetching the data as JSON and processing the result using jQuery, that injects the final HTML into place via Jinja2 templating.

Plotly was used to generated the chart that shows user activity times based on analysing most recent 100 Tweets.

Application styling and layout were done using Bootstrap and some custom CSS.

Some additional jQuery was used for things like character counter for the Tweet window or fade out for alerts.

Podłoga In-Development

Helper application for the daily management of the company I currently work for.

Key features:
  • Tracking the Start & End of work time for workers including the customer for whom the work was done
  • Keeping a list of customers, including data necessary for written protocols
  • Keeping track of material orders assigned to customers (including delivery date)

Technology outline:

Application backend is written using Django. Database is constructed using Django's built in models and set to use PostgreSQL as the engine.

Frontend is built with the help of Bootstrap and Django's templating language.

Most of the application content is provided dynamically via queries to database, whose results are being injected into the DOM using Django's built-in templating language.

Users are required to log in to use the application. Office admin staff is given "Staff" privileges, that allow them to check times logged in by on-site workers, edit customer details and add material orders.

Application is currently hosted on AWS for testing purposes.
